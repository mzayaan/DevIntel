## Project Overview

**DevIntel** is a full-stack developer news aggregator PWA. A Node.js/Express server proxies all external APIs (Dev.to, GitHub, Hacker News) with server-side caching and rate limiting. The frontend is vanilla JS + Tailwind CSS served statically by the same Express server, installable as a PWA on Android and iOS.

### Request Flow

```
Browser â†’ GET /api/news  â†’  Express (rateLimit â†’ cache check â†’ Dev.to fetch â†’ cache store) â†’ JSON
Browser â†’ GET /          â†’  Express serves client/index.html
Browser â†’ GET /css/styles.css â†’ Express serves compiled Tailwind CSS
```

### Key Architecture Decisions

**No bundler**: Four `<script>` tags load in order (`config.js` â†’ `api.js` â†’ `ui.js` â†’ `app.js`). They share the global scope intentionally. CommonJS `module.exports` guards at the bottom of `api.js` and `ui.js` enable direct `require()` in Jest without breaking browser behavior.

**Three-layer caching**: Server in-memory (Map + TTL) â†’ Client localStorage (10-min TTL) â†’ Service Worker cache. This means content is available offline and the GitHub rate limit (60/hr unauthenticated, 5000/hr with GITHUB_TOKEN) is shared across all clients through the server proxy.

**Dark mode**: Implemented via Tailwind's `class` strategy. `applyTheme(theme)` in `app.js` toggles `dark` on `<html>`. Reads `prefers-color-scheme` on first visit; persists choice in `localStorage.devintelTheme`.

**PWA icons**: Generated programmatically by `scripts/generate-icons.js` using `sharp`. Never edit the PNG files manually â€” edit the SVG template in the script and re-run `npm run build:icons`.

**Service Worker**: `client/pwa/service-worker.js` uses pathname-based routing: `/api/*` â†’ network-first, everything else â†’ cache-first. Cache name is `devintel-static-v2` / `devintel-api-v2` â€” bump these strings when deploying breaking changes.

### Environment Variables

See `.env.example`. The most important one is `GITHUB_TOKEN` â€” without it the GitHub API is limited to 60 req/hr across all users. Set `NODE_ENV=production` when deploying.

### Testing Notes

- Server tests (`server.test.js`) mock `global.fetch` via `jest.spyOn` because Node 18+ native fetch bypasses nock
- Server tests require `@jest-environment node` pragma (not jsdom)
- `ui.js` and `api.js` export functions via `if (typeof module !== 'undefined') module.exports = {...}` for testability
- E2E tests use Playwright's `webServer` config to auto-start the Express server
- Theme E2E tests call `applyTheme()` directly (sidebar button is behind mobile menu)
